# Access Control Policy

Organization: aLRELEVANT REGULATORS
Version: 1.0
Date: January 30, 2026
Owner: Security

## Purpose
Ensure access to PHI is restricted to authorized users and systems based on least privilege.

## Scope
All CareDroid systems, services, and personnel with access to PHI.

## Policy
- Unique user IDs are required for all users.
- MFA is required for administrative accounts and privileged roles.
- RBAC is enforced for all application endpoints handling PHI.
- Access requests require approval and documented justification.
- Access is reviewed quarterly and revoked upon role change or termination.

## User Access Lifecycle
1. Request: Manager submits access request.
2. Approval: Security and Compliance approval required.
3. Provisioning: Access granted based on role.
4. Review: Quarterly review of access rights.
5. Deprovisioning: Access removed immediately upon termination.

## Monitoring and Logging
- All access to PHI is logged in audit trails.
- Failed access attempts are reviewed weekly.

## Exceptions
Exceptions require documented approval from Security and Compliance.
